                    1. INTRODUCTION TO NAIVE BAYES
================================================================================

    ┌─────────────────────────────────────────────────────────────────────┐
    │                                                                     │
    │                    NAIVE BAYES                                      │
    │                    ────────────                                      │
    │                                                                     │
    │   A supervised learning algorithm based on Bayes theorem            │
    │   with "naive" assumption of feature independence.            │
    │                                                                     │
    └─────────────────────────────────────────────────────────────────────┘

KEY CHARACTERISTICS:

    ┌─────────────────────────────────────────────────────────────────────┐
    │  ✓ FAST AND SIMPLE                                                │
    │    Very fast training and prediction                                │
    │                                                                     │
    │  ✓ WORKS WELL WITH SMALL DATA                                      │
    │    Can learn quickly with minimal training data                    │
    │                                                                     │
    │  ✓ HANDLES HIGH DIMENSIONALITY                                  │
    │    Works well with many features                               │
    │                                                                     │
    │  ✓ ROBUST TO IRRELEVANT FEATURES                                │
    │    Not affected by irrelevant features                      │
    │                                                                     │
    │  ✓ PROBABILISTIC OUTPUT                                         │
    │    Provides probability estimates                             │
    │                                                                     │
    │  ✓ ASSUMES FEATURE INDEPENDENCE (NAIVE!)                        │
    │    May not hold in real data - hence "naive"                    │
    └─────────────────────────────────────────────────────────────────────┘

================================================================================
                    2. BAYES THEOREM
================================================================================

    ┌─────────────────────────────────────────────────────────────────────┐
    │                 Bayes Theorem:                                   │
    │                                                                     │
    │   P(y|X) = P(X|y) × P(y) / P(X)                                 │
    │                                                                     │
    │   Where:                                                          │
    │   • P(y|X) = posterior probability (what we want)                │
    │   • P(X|y) = likelihood (evidence given class)                   │
    │   • P(y) = prior probability (class probability)              │
    │   • P(X) = marginal probability (normalizing constant)            │
    │                                                                     │
    └─────────────────────────────────────────────────────────────────────┘

EXAMPLE:

    Medical diagnosis for disease D given test result T
    
    P(D|T) = P(T|D) × P(D) / P(T)
    
    P(D) = 0.01 (1% have disease)
    P(T|D) = 0.95 (test positive when disease present)
    P(T|not D) = 0.05 (false positive)
    
    P(T) = 0.95×0.01 + 0.05×0.99 = 0.059
    
    P(D|T) = 0.95×0.01 / 0.059 = 0.161
    
    Only 16% chance even with "accurate" test!

================================================================================
                    3. NAIVE ASSUMPTION
================================================================================

    ┌─────────────────────────────────────────────────────────────────────┐
    │                                                                     │
    │   NAIVE ASSUMPTION:                                                 │
    │   ────────────────                                                 │
    │                                                                     │
    │   Features are conditionally independent given class:           │
    │                                                                     │
    │   P(X|y) = P(x₁|y) × P(x₂|y) × ... × P(xₙ|y)                      │
    │                                                                     │
    │   Instead of computing joint probability, compute                │
    │   product of individual feature probabilities!                  │
    │                                                                     │
    └─────────────────────────────────────────────────────────────────────┘

WHY IT WORKS (EVEN THOUGH NAIVE):

    1. Even though independence is violated, the algorithm
       still classifies well because we're ranking classes.
    
    2. We're computing: argmax P(y|X) = argmax P(X|y) × P(y)
    
    3. The product approximation is good enough for ranking.
    
    4. Even if probabilities are wrong, ranking often correct.

================================================================================
                    4. TYPES OF NAIVE BAYES
================================================================================

4.1 GAUSSIAN NAIVE BAYES
──���───────────────────────

    ┌─────────────────────────────────────────────────────────────────────┐
    │                                                                     │
    │   For continuous features assumed normally distributed:            │
    │                                                                     │
    │   P(xᵢ|y) = (1/√(2πσ²)) × exp(-(xᵢ-μ)² / (2σ²))                    │
    │                                                                     │
    │   Parameters estimated from training data:                          │
    │   • μ = mean of feature for class y                               │
    │   • σ² = variance of feature for class y                           │
    │                                                                     │
    │   Best for: Continuous data (e.g., Iris, height)                   │
    │                                                                     │
    └─────────────────────────────────────────────────────────────────────┘

4.2 MULTINOMIAL NAIVE BAYES
───────────────────────────

    ┌─────────────────────────────────────────────────────────────────────┐
    │
    │   For discrete count data (e.g., word counts in text):               │
    │
    │   P(xᵢ|y) = (countᵢ + α) / (Σ count + α×|V|)                     │
    │
    │   Where:                                                          │
    │   • countᵢ = count of feature i in class y documents             │
    │   • |V| = vocabulary size                                        │
    │   • α = smoothing parameter (usually 1)                         │
    │
    │   Best for: Text classification, document categorization            │
    │
    └─────────────────────────────────────────────────────────────────────┘

4.3 BERNOULLI NAIVE BAYES
──────────────────────────

    ┌─────────────────────────────────────────────────────────────────────┐
    │                                                                     │
    │   For binary features (presence/absence):                           │
    │                                                                     │
    │   If xᵢ = 1: P(xᵢ=1|y) = pᵢ                                       │
    │   If xᵢ = 0: P(xᵢ=0|y) = 1 - pᵢ                                   │
    │                                                                     │
    │   Best for: Binary/multiclass text classification                   │
    │                                                                     │
    └─────────────────────────────────────────────────────────────────────┘

4.4 COMPLEMENT NAIVE BAYES
────────────────────────────

    ┌────────────────────────────────────────���─���──────────────────────────┐
    │                                                                     │
    │   Addresses severe assumption violation for imbalanced data:           │
    │                                                                     │
    │   Uses complement class statistics:                                │
    │   P(xᵢ|y) computed from complement class samples                   │
    │                                                                     │
    │   Better for imbalanced datasets                                   │
    │                                                                     │
    └─────────────────────────────────────────────────────────────────────┘

COMPARISON:

┌────────────────┬────────────────────────────────────────────────────────────┐
│ Type           │ Use Case                                          │
├───────────────┼──────────────────────────────────────────────────────────┤
│ Gaussian      │ Continuous data (measurements)                   │
│ Multinomial   │ Text/count data (word frequencies)           │
│ Bernoulli     │ Binary features (present/absent)              │
│ Complement   │ Imbalanced datasets                           │
└───────────────┴──────────────────────────────────────────────────┘

================================================================================
                    5. MATHEMATICAL DERIVATION
================================================================================

FOR CLASSIFICATION:

    ┌─────────────────────────────────────────────────────────────────────┐
    │                                                                     │
    │   PREDICTION:                                                       │
    │   argmax_y P(y|X)                                                   │
    │                                                                     │
    │   = argmax_y P(X|y) × P(y) / P(X)                                    │
    │                                                                     │
    │   Using naive assumption:                                          │
    │   = argmax_y [Πᵢ P(xᵢ|y)] × P(y)                                   │
    │                                                                     │
    │   Taking log:                                                     │
    │   = argmax_y [Σᵢ log P(xᵢ|y) + log P(y)]                         │
    │                                                                     │
    │   This is numerically more stable!                                 │
    │                                                                     │
    └─────────────────────────────────────────────────────────────────────┘

EXAMPLE:

    Classes: spam (y=1), not spam (y=0)
    Features: contains "free" (x₁=0/1), contains "money" (x₂=0/1)
    
    Training data shows:
    P(y=1) = 0.3, P(y=0) = 0.7
    
    For email with x₁=1, x₂=1:
    P(x₁=1|y=1) = 0.8, P(x₁=1|y=0) = 0.1
    P(x₂=1|y=1) = 0.6, P(x₂=1|y=0) = 0.05
    
    P(X|y=1) × P(y=1) = 0.8×0.6 × 0.3 = 0.144
    P(X|y=0) × P(y=0) = 0.1×0.05 × 0.7 = 0.0035
    
    Predicted: SPAM (higher probability)

================================================================================
                    6. LAPLACE SMOOTHING
================================================================================

    ┌─────────────────────────────────────────────────────────────────────┐
    │                                                                     │
    │   PROBLEM: If feature never appears in training for a class:            │
    │   P(xᵢ|y) × product = 0 → Zero probability!                         │
    │                                                                     │
    │   SOLUTION: Laplace (additive) smoothing:                              │
    │                                                                     │
    │   P(xᵢ|y) = (count(xᵢ,y) + α) / (count(y) + α×k)                 │
    │                                                                     │
    │   Where:                                                           │
    │   • α = smoothing parameter (usually 1)                         │
    │   • k = number of possible values for feature                       │
    │                                                                     │
    └─────────────────────────────────────────────────────────────────────┘

================================================================================
EXAMPLE 1: Gaussian Naive Bayes - Iris
================================================================================

from sklearn.naive_bayes import GaussianNB
from sklearn.datasets import load_iris
from sklearn.model_selection import train_test_split
from sklearn.metrics import accuracy_score, classification_report

print("="*70)
print("EXAMPLE 1: Gaussian Naive Bayes - Iris Dataset")
print("="*70)

iris = load_iris()
X, y = iris.data, iris.target

print(f"\nDataset: Iris ({X.shape[0]} samples, {X.shape[1]} features)")

X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

clf = GaussianNB()
clf.fit(X_train, y_train)

y_pred = clf.predict(X_test)

print(f"\n📊 Accuracy: {accuracy_score(y_test, y_pred):.4f}")
print(f"\n📋 Classification Report:")
print(classification_report(y_test, y_pred))

print(f"\n📊 Class Priors:")
print(f"   Class 0: {clf.class_prior_[0]:.4f}")
print(f"   Class 1: {clf.class_prior_[1]:.4f}")
print(f"   Class 2: {clf.class_prior_[2]:.4f}")

print(f"\n📊 Mean for each feature (class 0):")
print(f"   {clf.theta_[0]}")

================================================================================
EXAMPLE 2: Multinomial Naive Bayes - Text
================================================================================

from sklearn.naive_bayes import MultinomialNB
from sklearn.feature_extraction.text import CountVectorizer

print("\n" + "="*70)
print("EXAMPLE 2: Multinomial Naive Bayes - Text Classification")
print("="*70)

texts = [
    "Chinese Beijing Chinese",
    "Chinese Chinese Shanghai",
    "Chinese Macao",
    "Tokyo Japan Chinese",
    "Japanese",
    "Tokyo Yokohama"
]
labels = [1, 1, 1, 0, 0, 0]  # 1=China, 0=Japan

vectorizer = CountVectorizer()
X = vectorizer.fit_transform(texts)

print(f"\nFeatures: {vectorizer.get_feature_names()}")
print(f"Feature matrix shape: {X.shape}")

clf = MultinomialNB()
clf.fit(X, labels)

# Predict new text
new_text = ["Chinese Tokyo"]
new_X = vectorizer.transform(new_text)
prediction = clf.predict(new_X)

print(f"\n📊 Prediction for '{new_text[0]}': {'China' if prediction[0] else 'Japan'}")

# Probability
proba = clf.predict_proba(new_X)
print(f"   Probabilities: Japan={proba[0][0]:.4f}, China={proba[0][1]:.4f}")

from sklearn.naive_bayes import ComplementNB, MultinomialNB

# Compare

                  ("Complement", ComplementNB())]:
    scores = cross_val_score(clf, X, y, cv=5)
    print(f"   {name:<12}: {scores.mean():.4f} (+/- {scores.std():.4f})")

EXAMPLE 5: Gaussian vs Multinomial
================================================================================

from sklearn.naive_bayes import GaussianNB, MultinomialNB
from sklearn.datasets import load_iris
from sklearn.model_selection import cross_val_score
from sklearn.preprocessing import StandardScaler
import numpy as np

print("\n" + "="*70)
print("EXAMPLE 5: Gaussian vs Multinomial - Iris")
print("="*70)

iris = load_iris()
X, y = iris.data, iris.target

# Gaussian
clf_g = GaussianNB()
scores_g = cross_val_score(clf_g, X, y, cv=5)

# Multinomial (need non-negative - use binned)
X_binned = np.digitize(X, np.percentile(X, [25, 50, 75]))
clf_m = MultinomialNB()
scores_m = cross_val_score(clf_m, X_binned, y, cv=5)

print(f"\n📊 5-Fold Cross-Validation:")
print(f"   Gaussian:   {scores_g.mean():.4f} (+/- {scores_g.std():.4f})")
print(f"   Multinomial: {scores_m.mean():.4f} (+/- {scores_m.std():.4f})")

print("\nNote: Gaussian works better for continuous Iris data")

from sklearn.naive_bayes import GaussianNB
from sklearn.datasets import load_iris
from sklearn.model_selection import train_test_split

print("\n" + "="*70)
print("EXAMPLE 6: Probabilistic Output")
print("="*70)

iris = load_iris()
X, y = iris.data, iris.target

X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

clf = GaussianNB()

# Get probabilities
sample = X_test[:1]
pred = clf.predict(sample)[0]
proba = clf.predict_proba(sample)[0]

print(f"\n📊 Sample: {sample[0]}")
print(f"   Predicted class: {pred}")
print(f"   Probabilities:")
for i, p in enumerate(proba):
    bar = "█" * int(p * 20)
    print(f"      {iris.target_names[i]:<12}: {p:.4f} {bar}")

    ┌─────────────────────────────────────────────────────────────────────┐
    │                     NAIVE BAYES SUMMARY                           │
    ├─────────────────────────────────────────────────────────────────────┤
    │                                                                     │
    │   WHAT IT DOES:                                                 │
    │   Uses Bayes theorem with naive independence assumption        │
    │   to compute P(y|X) for classification                        │
    │                                                                     │
    │   TYPES:                                                       │
    │   • Gaussian: Continuous features (normal distribution)       │
    │   • Multinomial: Count data (text)                             │
    │   • Bernoulli: Binary features                                │
    │   • Complement: Imbalanced data                             │
    │                                                                     │
    │   PROS:                                                       │
    │   ✓ Very fast (no parameters to learn after priors)            │
    │   ✓ Works well with small data                                   │
    │   ✓ Handles high dimensionality                             │
    │   ✓ Provides probability estimates                          │
    │   ✓ Robust to irrelevant features                            │
    │                                                                     │
    │   CONS:                                                       │
    │   ✗ Naive assumption almost always violated                   │
    │   ✗ Can have zero probability issues                        │
    │   ✗ Not as accurate as other classifiers for some data          │
    │                                                                     │
    │   BEST FOR:                                                   │
    │   • Text classification (spam, sentiment)                   │
    │   • Quick baseline model                                     │
    │   • Multi-class problems                                    │
    │   • When interpretability needed (probability output)          │
    │                                                                     │
    └─────────────────────────────────────────────────────────────────────┘
